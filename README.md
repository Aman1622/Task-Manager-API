# Task Manager API

A powerful RESTful API for managing tasks, built with Django and Django REST Framework.

## Features

*   **User Registration and Authentication:** Secure JSON Web Token (JWT) based authentication.
*   **Role-Based Access Control:** Separate permissions for Admin (global access) and Regular Users (own tasks only).
*   **CRUD Operations:** Complete Create, Read, Update, Delete functionality for tasks.
*   **Pagination:** Smooth and quick navigation, displaying 10 tasks per page by default.
*   **Filtering and Searching:** Find exactly what you need with advanced search and boolean filters (completed status).
*   **Automated API Documentation:** Fully interactive API guides generated by Swagger UI and ReDoc using OpenAPI 3.0 schema.
*   **Automated Testing:** Unit test suites powered by pytest and pytest-django for assuring system correctness.

## Installation & Setup

1. **Clone the repository:**
   \`\`\`bash
   git clone <https://github.com/Aman1622/Task-Manager-API>
   cd TaskAPI
   \`\`\`

2. **Create and Activate Virtual Environment:**
   \`\`\`bash
   python3 -m venv venv
   source venv/bin/activate  # On macOS/Linux
   # venv\Scripts\activate  # On Windows
   \`\`\`

3. **Install Dependencies:**
   \`\`\`bash
   pip install -r requirements.txt
   \`\`\`

4. **Run Migrations:**
   \`\`\`bash
   python manage.py makemigrations
   python manage.py migrate
   \`\`\`

5. **Start the Development Server:**
   \`\`\`bash
   python manage.py runserver
   \`\`\`

The API will be available at: \`http://127.0.0.1:8000/\`

## API Documentation

Interactive API documentation is automatically generated. After starting the server, go to:

*   **Swagger UI:** \`http://127.0.0.1:8000/api/schema/swagger-ui/\`
*   **ReDoc:** \`http://127.0.0.1:8000/api/schema/redoc/\`

## Authentication

Authentication is required for most endpoints.

1.  **Register a new user:**
    \`POST /api/auth/register/\`
    \`\`\`json
    {
      "username": "johndoe",
      "email": "john@example.com",
      "password": "securepassword",
      "role": "user" // Or "admin"
    }
    \`\`\`

2.  **Login to get a token:**
    \`POST /api/auth/login/\`
    \`\`\`json
    {
      "username": "johndoe",
      "password": "securepassword"
    }
    \`\`\`
    Response will contain \`access\` and \`refresh\` tokens.

3.  **Include token in headers for protected routes:**
    \`Authorization: Bearer <your_access_token>\`

## API Endpoints Overview

**Authentication**
*   \`POST /api/auth/register/\` - Register a new user
*   \`POST /api/auth/login/\` - Get JWT pair
*   \`POST /api/auth/login/refresh/\` - Refresh access token
*   \`GET/PUT /api/auth/profile/\` - Manage current user profile

**Tasks**
*   \`GET /api/tasks/\` - List tasks (Filtered by user unless admin)
    *   Query params: \`?page=1\`, \`?completed=true\`, \`?search=keyword\`, \`?ordering=-created_at\`
*   \`POST /api/tasks/\` - Create a new task
*   \`GET /api/tasks/{id}/\` - Retrieve a task
*   \`PUT /api/tasks/{id}/\` - Update a task fully
*   \`PATCH /api/tasks/{id}/\` - Update a task partially
*   \`DELETE /api/tasks/{id}/\` - Delete a task
